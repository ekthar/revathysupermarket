# PWA premium fixes: status bar, layout shifts, animations, navigation speed

This change addresses five production-feel issues in a Next.js 15 PWA: dynamic theme-color sync for the iOS status bar, responsive layout overflow/CLS fixes, loading skeletons for bottom-nav routes, a Vaul-based bottom sheet with standardized micro-interactions, and aggressive touchstart prefetching for tab switches. The approach is pragmatic—mostly CSS and component-level fixes with one new dependency (vaul).

**Watch for:** ThemeColorSync may race with Next.js's own `<meta name="theme-color">` rendering when the page has multiple theme-color meta tags (confirmed). The Vaul BottomSheet omits `Drawer.Title` when no `title` prop is passed, producing an accessibility warning and missing ARIA label (confirmed). The `onTouchStart` prefetch handler fires on every touch, including scroll-through touches on mobile, triggering unnecessary network requests (likely).

## High-level view

The ThemeColorSync component uses `usePathname` and a MutationObserver on the `<html>` class list to keep the iOS status bar color in sync with route and dark-mode state. It works for single-meta-tag setups, but the layout exports *two* theme-color meta tags (one per color scheme media query), so `querySelector('meta[name="theme-color"]')` will only update the first one, leaving the second stale.

The Vaul migration preserves the existing BottomSheet API surface (`open`, `onClose`, `title`, `children`) but conditionally renders `Drawer.Title` only when `title` is provided, leaving sheets without a title lacking an accessible name.

The nav prefetch strategy adds `router.prefetch()` on `touchstart` and `mouseenter` to warm route payloads ahead of tap. Combined with the new loading skeletons, this gives near-instant perceived navigation. The concern is that touchstart fires on scroll gestures too, so passive scrolling through the nav area will fire prefetches.

The LiveOrderBanner now animates height from 0 to auto via framer-motion instead of conditionally mounting, eliminating the layout jump. Framer-motion measures natural height on mount; if banner content changes height dynamically after that measurement, the container won't re-adapt without a remount.

`lib/motion.ts` defines `springPresets`, `tapScale`, and `pageTransition`, but no component in this diff imports them. Components still hardcode inline transition values. The consolidation goal is only half-delivered.

1. **Multiple theme-color meta tags** — `layout.tsx` exports two `<meta name="theme-color">` tags (one per `prefers-color-scheme` media). `querySelector` only updates the first. Either consolidate to a single meta tag in the layout viewport export or update all matching tags in `ThemeColorSync`.

## ThemeColorSync and dual meta tags

The `layout.tsx` viewport export produces two `<meta name="theme-color">` elements—one with `media="(prefers-color-scheme: light)"` and one for dark. The `ThemeColorSync` component calls:

```ts
document.querySelector('meta[name="theme-color"]')
```

This returns only the first match. On a device where the OS is in dark mode, the browser respects the second meta tag (the one with the dark media query), but `ThemeColorSync` is updating the first one—which the browser ignores because its media query doesn't match. The result: dynamic route colors work only when the OS color scheme matches the first meta tag's media attribute.

Fix: use `querySelectorAll` and update all theme-color meta tags, or remove the media attribute from the viewport export (since the component now manages the color dynamically based on the app's own theme class, not the OS preference).

## Vaul accessibility gap

When `BottomSheet` is called without a `title` prop (which the interface explicitly allows as optional), `Drawer.Title` is not rendered. Vaul's `Drawer.Content` renders a `role="dialog"` element, and dialogs require an accessible name per WCAG. Vaul logs a console warning about this. The product-grid filter sheet does pass a title, but any future caller that omits it will produce an unlabelled dialog.

Fix: always render `<Drawer.Title>` with a visually-hidden fallback (e.g., "Dialog") when no title is provided.

## Prefetch and touchstart semantics

On mobile, `touchstart` fires at the beginning of any touch, including scroll gestures. If the user's thumb rests near the bottom nav while scrolling content, each scroll will trigger `router.prefetch()` for whatever tab is under the thumb. Next.js deduplicates identical prefetch calls within a session, so repeated calls to the same route are cheap after the first. But across four tabs, initial scrolls near the nav bar could fire four prefetches simultaneously on first visit.

The standard pattern to avoid this is to track `onTouchMove` and cancel the prefetch intent, or to use `onPointerDown` with a small setTimeout that's cleared on `onPointerMove`.

## Shared motion presets: defined but unused

`lib/motion.ts` exports `springPresets`, `tapScale`, and `pageTransition`, but no component in this diff imports them. The cart page uses `{ type: "spring", stiffness: 300, damping: 30 }` inline (matches `springPresets.default`). The product card uses `{ stiffness: 400, damping: 25 }` (matches `springPresets.snappy`). The mobile nav uses `{ stiffness: 500, damping: 25 }` (matches nothing in the presets). The file exists as a reference but doesn't reduce duplication yet.
